# CONTRIBUTING

Thank you for your interest in contributing to ACE! We welcome contributions from the community, whether it's reporting bugs, suggesting new features, or contributing code. To ensure a smooth contribution process, please follow the guidelines outlined below.

## Code of Conduct
We expect all contributors to adhere to our Code of Conduct, which promotes a welcoming and inclusive environment for everyone. Please read the [Code of Conduct](CODE_OF_CONDUCT.md) before contributing.

## Issue Reporting
If you encounter a bug or have a feature request, please open an issue on our GitHub repository. When reporting an issue, please provide as much detail as possible, including steps to reproduce the issue, expected behavior, and any relevant screenshots or logs.

## Pull Requests
If you would like to contribute code, please fork the repository and create a new branch for your changes. Once you have made your changes, submit a pull request with a clear description of what you have done and why it is necessary. Please ensure that your code follows our coding standards and includes appropriate tests.

## Code Style
We use `black` for code formatting. Please ensure that your code is formatted according to `black` before submitting a pull request. You can run `black .` in the project directory to format your code.

## Testing
All new features and bug fixes should include appropriate tests. Tests should be placed in the `tests` directory and follow the existing structure. Please ensure that all tests pass before submitting your pull request.

## Documentation
If your contribution includes changes to the functionality of ACE, please also update the documentation accordingly. This includes updating the README, adding new sections to the documentation, and ensuring that any new tools or features are well-documented.

## License
Please ensure that your contributions do not include any code that is not compatible with our license. By contributing to ACE, you agree that your contributions will be licensed under the same terms as the project.
